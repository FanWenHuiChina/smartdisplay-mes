## 已知限制（简化版）

### 1. 密码未加密

**当前实现**：
```java
if (!user.getPassword().equals(request.getPassword())) {
    throw new BusinessException("用户名或密码错误");
}
```

**商用标准**：
- 应该使用BCrypt加密
- 数据库存储密码哈希值
- 登录时验证哈希

**改进方案**：
```java
// 添加Spring Security依赖
// 使用BCryptPasswordEncoder
if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
    throw new BusinessException("用户名或密码错误");
}
```

### 2. 没有Token刷新机制

**当前实现**：
- Token 24小时后失效
- 失效后需要重新登录

**商用标准**：
- AccessToken（短期，如15分钟）
- RefreshToken（长期，如7天）
- AccessToken过期时用RefreshToken刷新

### 3. 没有RBAC权限控制

**当前实现**：
- 只有role字段
- 没有权限验证

**商用标准**：
- 角色-权限表（sys_role, sys_permission, sys_role_permission）
- 接口权限注解（@RequiresPermission）
- 按钮级别权限控制

### 4. Token只存localStorage

**安全风险**：
- XSS攻击可读取localStorage
- 浏览器关闭后不会自动清除

**更安全的方案**：
- HttpOnly Cookie存储（防XSS）
- 或使用SessionStorage（浏览器关闭自动清除）
