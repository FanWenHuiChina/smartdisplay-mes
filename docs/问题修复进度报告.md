   ```yaml

     servlet:

         charset: UTF-8
         enabled: true
         force: true
   ```

2. **前端request.js**：

**预期效果**：
- Track Out弹窗输入中文备注不再报错
- 所有中文数据正常传输

### ✅ P0-3: 工序/设备选项动态加载（已完成）

**修改内容**：
1. **新增后端接口**：
   - `MasterDataController` - 主数据管理接口
   - `GET /api/master-data/process-steps` - 获取工序列表
   - `GET /api/master-data/equipments` - 获取设备列表
   - `MasterDataService` - 查询ProcessStep和Equipment表

2. **前端API封装**：
   - 新增`api/masterdata.js`
   - `getProcessSteps()`和`getEquipments()`方法

3. **TrackInDialog改造**：
   - 去除硬编码的工序/设备选项
   - 从后端动态加载选项数据
   - 使用`v-for`渲染下拉选项
   - `onMounted`时调用`loadOptions()`

**优势**：
- 工序/设备数据可以从数据库动态维护
- 新增设备无需修改前端代码
- 符合商用标准的数据管理方式
